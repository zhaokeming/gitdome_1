## Redis应用场景（后端配合）

虽然前端不直接使用Redis，但项目设计与后端的Redis应用紧密结合：

1. **String** - Token黑名单（前端401处理）
2. **Hash** - 缓存公告（前端快速加载）
3. **Set** - 工单状态集合（前端筛选）
4. **ZSet** - 优先级队列（管理员高优先级工单展示）

## 注意事项

1. **后端依赖** - 需要后端服务运行在8080端口
2. **跨域问题** - 开发时Vite已配置代理，生产环境需Nginx配置
3. **Token过期** - 已实现401自动跳转登录
4. **浏览器兼容** - 建议使用Chrome、Firefox等现代浏览器

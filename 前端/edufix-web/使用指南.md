# EduFix 前端使用指南

## 项目概述

这是一个基于Vue3 + Element Plus开发的校园/社区综合服务与报修工单平台前端项目。

## 快速启动

### 1. 确保后端已启动

首先确保后端服务正在运行（默认端口：8080）

### 2. 安装依赖

```bash
npm install
```

### 3. 启动开发服务器

```bash
npm run dev
```

浏览器会自动打开 http://localhost:3000

### 4. 测试账号

您可以在注册页面创建新账号，或者使用以下测试账号（需要后端有对应数据）：

- 密码: 123456

**管理员：**

## 功能演示流程

### 普通用户流程

1. **注册/登录**
   - 访问 http://localhost:3000/register 注册账号
   - 或使用已有账号登录

2. **创建工单**
   - 登录后点击"创建工单"
   - 选择工单类型（报修/建议/失物招领）
   - 填写详细信息并提交

3. **查看工单**
   - 在"我的工单"页面查看所有工单
   - 点击详情查看工单进度

4. **评价服务**
   - 工单完成后，在详情页进行评价
   - 可以评分、选择标签、填写评价内容

5. **查看公告**
   - 浏览管理员发布的公告信息

### 管理员流程

1. **登录管理后台**
   - 使用管理员账号登录
   - 自动跳转到管理后台

2. **查看统计数据**
   - Dashboard显示工单统计信息
   - 查看高优先级工单

3. **工单派单**
   - 进入"工单派单"页面
   - 查看待处理工单列表
   - 选择合适的维修员进行派单

4. **发布公告**
   - 进入"公告管理"页面
   - 点击"发布公告"
   - 填写标题和内容并发布

5. **查看所有工单**
   - 可以查看所有用户提交的工单
   - 支持按状态和类型筛选

### 维修员流程

1. **登录工作台**
   - 使用维修员账号登录
   - 自动跳转到工作台

2. **查看分配的工单**
   - 在Dashboard查看工单统计
   - 在列表中查看分配的工单

3. **处理工单**
   - 点击"开始处理"改变工单状态
   - 完成后点击"完成"

4. **查看工单详情**
   - 查看用户提交的具体问题
   - 查看位置和联系方式

## 常见问题

### 1. 登录后跳转到错误页面

检查：
- 后端服务是否正常运行
- API地址配置是否正确（src/utils/request.js）
- 账号角色是否正确设置

### 2. 无法创建工单

检查：
- 是否已登录
- 表单是否填写完整
- 后端接口是否正常

### 3. 看不到某些菜单

不同角色看到的菜单不同：
- 普通用户：只能看到自己的工单相关菜单
- 管理员：可以看到所有管理功能
- 维修员：只能看到工作台和自己的工单

### 4. 样式显示异常

尝试：
- 清除浏览器缓存
- 重新运行 `npm install`
- 检查Element Plus是否正确安装

## 开发提示

### 修改API地址

编辑 `src/utils/request.js`：

```javascript
const request = axios.create({
  baseURL: 'http://localhost:8080/api', // 修改这里
  timeout: 10000
})
```

### 添加新功能

1. 在 `src/api/` 下创建API文件
2. 在 `src/views/` 下创建页面组件
3. 在 `src/router/index.js` 中添加路由
4. 在 `src/layout/MainLayout.vue` 中添加菜单项

### 调试技巧

- 打开浏览器开发者工具查看网络请求
- 在Console中查看错误信息
- 使用Vue Devtools调试组件状态

## 性能优化建议

1. **代码分割** - 已实现路由级别的懒加载
2. **组件复用** - 提取公共组件
3. **图片优化** - 使用适当的图片格式和大小
4. **缓存策略** - 合理使用浏览器缓存

## 部署

### 构建生产版本

```bash
npm run build
```

构建后的文件在 `dist/` 目录

### 部署到服务器

将 `dist/` 目录的内容复制到Web服务器（如Nginx、Apache）的根目录

### Nginx配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API代理（如果需要）
    location /api/ {
        proxy_pass http://localhost:8080;
    }
}
```
